官方文档

指令
init
$ hexo init [folder]
新建一个网站。 如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站。

本命令相当于执行了以下几步：

Git clone hexo-starter 和 hexo-theme-landscape 主题到当前目录或指定目录。
使用 Yarn 1、pnpm 或 npm 包管理器下载依赖（如有已安装多个，则列在前面的优先）。 npm 默认随 Node.js 安装。
new
$ hexo new [layout] <br>新建一篇文章。 如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。 Use the layout draft to create a draft. 如果标题包含空格的话，请使用引号括起来。</p> <p>选项 描述<br>-p, –path 文章的路径。 自定义文章的路径。<br>-r, –replace 如果存在的话，替换当前的文章。<br>-s, –slug 文章别名。 自定义文章的 URL。<br>默认情况下，Hexo 会使用文章的标题来决定文章文件的路径。 对于独立页面来说，Hexo 会创建一个以标题为名字的目录，并在目录中放置一个 index.md 文件。 你可以使用 –path 参数来覆盖上述行为、自行决定文件的目录：</p> <p>hexo new page –path about/me “About me”<br>以上命令会创建一个 source/about/me.md 文件，同时 Front Matter 中的 title 为 “About me”</p> <p>注意！ title 是必须指定的！ For example, this will not result in the behaviour you might expect:</p> <p>hexo new page –path about/me<br>此时 Hexo 会创建 source/_posts/about/me.md，同时 me.md 的 Front Matter 中的 title 为 “page”。 这是因为在上述命令中，hexo-cli 将 page 视为指定文章的标题、并采用默认的 layout。</p> <p>generate<br>$ hexo generate<br>生成静态文件。</p> <p>选项 描述<br>-d, –deploy Deploy after generation finishes<br>-w, –watch 监视文件变动<br>-b, –bail 生成过程中如果发生任何未处理的异常则抛出异常<br>-f, –force 强制重新生成<br>-c, –concurrency 要同时生成的文件的最大数量。 默认无限制<br>publish<br>$ hexo publish [layout] <filename><br>发表草稿。</p> <p>server<br>$ hexo server<br>启动服务器。 默认情况下，访问网址为： <a target="_blank" rel="noopener" href="http://localhost:4000/%E3%80%82">http://localhost:4000/。</a></p> <p>选项 描述<br>-p, –port 重设端口<br>-s, –static 只使用静态文件<br>-l, –log Enable logger. Override logger format.<br>deploy<br>$ hexo deploy<br>部署你的网站。</p> <p>选项 描述<br>-g, –generate Generate before deployment<br>render<br>$ hexo render <file1> [file2] …<br>渲染文件。</p> <p>选项 描述<br>-o, –output Output destination<br>migrate<br>$ hexo migrate <type><br>从其他博客系统 迁移内容。</p> <p>clean<br>$ hexo clean<br>清除缓存文件 (db.json) 和已生成的静态文件 (public)。</p> <p>list<br>$ hexo list <type><br>列出所有路由。</p> <p>version<br>$ hexo version<br>显示版本信息。</p> <p>config<br>$ hexo config [key] [value]<br>列出网站的配置（_config.yml）。 如果指定了 key，则只展示配置中对应 key 的值；如果同时指定了 key 和 value，则将配置中对应的 key 的值修改为 value。</p> <p>选项<br>安全模式<br>$ hexo –safe<br>在安全模式下，不会加载插件和脚本。 当您在安装新插件遭遇问题时，可以尝试以安全模式重新执行。</p> <p>调试模式<br>$ hexo –debug<br>在终端中显示调试信息并记录到 debug.log。 当您使用 Hexo 时遇到问题，可以尝试用调试模式重新执行一次。 如果您发现错误，请在 GitHub 上提出 issue。</p> <p>简洁模式<br>$ hexo –silent<br>Silences output to the terminal.</p> <p>自定义配置文件的路径<br>$ hexo –config custom.yml<br>自定义配置文件的路径，指定这个参数后将不再使用默认的 _config.yml。 还接受一个以逗号分隔的 JSON 或 YAML 配置文件列表（无空格），该列表将把这些文件合并为一个 _multiconfig.yml 文件。</p> <p>$ hexo –config custom.yml,custom2.json<br>显示草稿<br>$ hexo –draft<br>显示 source/_drafts 文件夹中的草稿文章。</p> <p>自定义 CWD<br>$ hexo –cwd /path/to/cwd<br>自定义当前工作目录（Current working directory）的路径。</p> <hr> <p>一、Hexo 常用命令</p> <p>hexo init：此命令用于初始化一个本地文件夹作为网站的根目录。通过运行 hexo init <folder>，您可以在指定文件夹中创建一个新的 Hexo 网站。</p> <p>hexo new：此命令用于新建一篇文章。通过运行 hexo new <title>，您可以创建一个新的 Markdown 文件，该文件将作为一篇新的博客文章。您还可以使用 hexo new page <pageName> 来新建一个页面。</p> <p>hexo generate：此命令用于生成静态文件。在编写完文章后，您需要运行 hexo generate 或简写为 hexo g 来将 Markdown 文件转换为 HTML 文件。这些 HTML 文件将被存储在 public 目录中。</p> <p>hexo server：此命令用于启动本地服务器，以便预览您的博客。通过运行 hexo server 或简写为 hexo s，您可以在本地浏览器中访问 <a target="_blank" rel="noopener" href="http://localhost:4000/">http://localhost:4000</a> 来查看您的博客。您还可以使用 -p 选项来指定服务器端口，使用 -i 选项来指定服务器 IP 地址。</p> <p>hexo deploy：此命令用于部署网站。在您的博客准备就绪后，您可以运行 hexo deploy 或简写为 hexo d 来将您的博客部署到远程服务器上。Hexo 支持多种部署方式，如 GitHub、GitLab 等。</p> <p>hexo clean：此命令用于清理生成的静态文件和缓存文件。在运行 hexo clean 后，public 目录中的所有文件将被删除。</p> <p>二、Hexo 命令缩写</p> <p>为了更快速地执行命令，Hexo 提供了一些命令缩写。例如，hexo new 可以简写为 hexo n，hexo generate 可以简写为 hexo g，hexo server 可以简写为 hexo s，hexo deploy 可以简写为 hexo d。</p> <p>三、Hexo 组合命令</p> <p>Hexo 还支持组合命令，以便更高效地执行任务。例如，hexo s -g 命令将同时启动本地服务器并生成静态文件，这样您可以在预览博客的同时确保生成的静态文件是最新的。同样，hexo d -g 命令将同时生成静态文件并部署到远程服务器。</p> </article><div class="post-copyright"><div class="post-copyright__author"><span class="post-copyright-meta"><i class="fas fa-circle-user fa-fw"></i>文章作者: </span><span
